# 笔记

  1. 动态值:export语句输出的接口，与其对应的值是动态绑定关系，即通过该接口，可以取到模块内部实时的值。
  2. 只读性(单向数据流): import命令输入的变量都是只读的，因为它的本质是输入接口。也就是说，不允许在加载模块的脚本里面，改写接口。 上面代码中，重写 d 的值就会报错，因为d是一个只读的接口。但是，如果d是一个对象，改写d的属性是允许的。不过，这种写法很难查错，建议凡是输入的变量，都当作完全只读，轻易不要改变它的属性。
